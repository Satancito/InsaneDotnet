# InsaneIO.Insane.Cryptography

Parent namespace: [InsaneIO.Insane](../Insane.md)

This namespace contains the concrete, reusable cryptography components exposed by the library.

Use these classes when you want a configured object with serialization support, rather than one-off static helper calls.

## Public types

- `Base32Encoder`
- `Base64Encoder`
- `HexEncoder`
- `ShaHasher`
- `HmacHasher`
- `ScryptHasher`
- `Argon2Hasher`
- `AesCbcEncryptor`
- `RsaKeyPair`
- `RsaEncryptor`

## Choosing between classes and extension methods

- Prefer the classes in this namespace when you want reusable configuration and JSON round-trips.
- Prefer `InsaneIO.Insane.Cryptography.Extensions` when you want direct functional calls without object state.
- Prefer `IEncoder.DeserializeDynamic`, `IHasher.DeserializeDynamic`, or `IEncryptor.DeserializeDynamic` when the concrete type is not known ahead of time.

## Encoders

### Base32Encoder

Represents Base32 encoding with configurable casing and padding output.

#### Configuration

- `ToLower` defaults to `false`
- `RemovePadding` defaults to `false`
- `DefaultInstance` is available through `IDefaultInstance<Base32Encoder>`

#### Methods

- `Encode(byte[] data)`
- `Encode(string data)` uses UTF-8
- `Decode(string data)` accepts trimmed Base32 input and supports either case
- `Serialize(...)` and `Deserialize(...)`

#### JSON shape

```json
{
  "TypeIdentifier": "Insane-Cryptography-Base32Encoder",
  "RemovePadding": false,
  "ToLower": false
}
```

#### Typical usage

```csharp
var encoder = new Base32Encoder
{
    RemovePadding = true,
    ToLower = true
};

string encoded = encoder.Encode("hello");
byte[] bytes = encoder.Decode(encoded);
string json = encoder.Serialize(indented: true);
IEncoder roundTrip = Base32Encoder.Deserialize(json);
```

### Base64Encoder

Represents Base64 encoding with line-break control, padding control, and variant selection.

#### Configuration

- `LineBreaksLength` defaults to `0`
- `RemovePadding` defaults to `false`
- `EncodingType` defaults to `Base64`
- constants:
  - `NoLineBreaks`
  - `MimeLineBreaksLength`
  - `PemLineBreaksLength`

#### Important behavior

- `Encode(...)` honors the configured variant.
- `Decode(...)` delegates to `DecodeFromBase64()`, which accepts standard, URL-safe, filename-safe, and URL-encoded Base64.

#### JSON shape

```json
{
  "TypeIdentifier": "Insane-Cryptography-Base64Encoder",
  "LineBreaksLength": 76,
  "RemovePadding": false,
  "EncodingType": 0
}
```

`EncodingType` is written as its numeric enum value.

#### Example

```csharp
var encoder = new Base64Encoder
{
    LineBreaksLength = Base64Encoder.MimeLineBreaksLength,
    EncodingType = Base64Encoding.Base64
};

string pemBody = encoder.Encode("payload");
byte[] raw = encoder.Decode(pemBody);
```

### HexEncoder

Represents hexadecimal encoding with configurable uppercase output.

#### Configuration

- `ToUpper` defaults to `false`
- `DefaultInstance` is available

#### Example

```csharp
var encoder = new HexEncoder { ToUpper = true };
string encoded = encoder.Encode("hello");
byte[] bytes = encoder.Decode(encoded);
```

## Hashers

### ShaHasher

Represents a configurable digest hasher with encoded output support.

#### Configuration

- `HashAlgorithm` defaults to `Sha512`
- `Encoder` defaults to `Base64Encoder.DefaultInstance`

#### Methods

- `Compute(byte[] data)`
- `Compute(string data)`
- `ComputeEncoded(...)`
- `Verify(...)`
- `VerifyEncoded(...)`

`Compute(string data)` uses UTF-8. `ComputeEncoded(...)` encodes the raw digest through the configured encoder.

#### JSON shape

```json
{
  "TypeIdentifier": "Insane-Cryptography-ShaHasher",
  "HashAlgorithm": 4,
  "Encoder": { ... }
}
```

#### Example

```csharp
var hasher = new ShaHasher
{
    HashAlgorithm = HashAlgorithm.Sha256,
    Encoder = HexEncoder.DefaultInstance
};

string digest = hasher.ComputeEncoded("payload");
bool ok = hasher.VerifyEncoded("payload", digest);
string json = hasher.Serialize();
IHasher restored = IHasher.DeserializeDynamic(json);
```

### HmacHasher

Represents a keyed HMAC configuration.

#### Configuration

- `HashAlgorithm` defaults to `Sha512`
- `Encoder` defaults to `Base64Encoder.DefaultInstance`
- `KeyBytes` and `KeyString` are alternative init-only ways to configure the secret
- an internal random key is generated by default when no key is supplied

#### Important property behavior

- `KeyString` is exposed through the configured encoder
- `KeyBytes` gives direct byte access

#### Example

```csharp
var hasher = new HmacHasher
{
    KeyString = "shared-secret",
    HashAlgorithm = HashAlgorithm.Sha384,
    Encoder = Base64Encoder.DefaultInstance
};

string mac = hasher.ComputeEncoded("payload");
bool valid = hasher.VerifyEncoded("payload", mac);
```

### ScryptHasher

Represents a stateful SCrypt configuration suitable for repeated derivation using the same parameters.

#### Configuration

- `SaltBytes` / `SaltString`
- `Iterations`
- `BlockSize`
- `Parallelism`
- `DerivedKeyLength`
- `Encoder`

Defaults come from `Constants`.

#### Example

```csharp
var hasher = new ScryptHasher
{
    SaltString = "salt",
    Iterations = 16384,
    BlockSize = 8,
    Parallelism = 1,
    DerivedKeyLength = 64,
    Encoder = Base64Encoder.DefaultInstance
};

string derived = hasher.ComputeEncoded("password");
```

### Argon2Hasher

Represents an Argon2 configuration with explicit salt, iteration, memory, and parallelism settings.

#### Configuration

- `SaltBytes` / `SaltString`
- `Iterations`
- `MemorySizeKiB`
- `DegreeOfParallelism`
- `DerivedKeyLength`
- `Argon2Variant`
- `Encoder`

Defaults:

- `Argon2Variant.Argon2id`
- size and work factors from `Constants`

#### Example

```csharp
var hasher = new Argon2Hasher
{
    SaltString = "salt",
    Iterations = 2,
    MemorySizeKiB = 16384,
    DegreeOfParallelism = 4,
    DerivedKeyLength = 64,
    Argon2Variant = Argon2Variant.Argon2id,
    Encoder = Base64Encoder.DefaultInstance
};

string derived = hasher.ComputeEncoded("password");
```

## Encryptors

### AesCbcEncryptor

Represents a reusable AES-CBC encryptor/decryptor.

#### Configuration

- `KeyBytes` / `KeyString`
- `Encoder`
- `Padding`

Important implementation details:

- the input key is normalized internally through `SHA512` and truncated to 32 bytes
- the IV is generated for each encryption
- the ciphertext format stores the IV at the end of the returned byte array

#### Methods

- `Encrypt(byte[] data)`
- `Encrypt(string data)`
- `EncryptEncoded(...)`
- `Decrypt(byte[] data)`
- `DecryptEncoded(string data)`

#### Example

```csharp
var encryptor = new AesCbcEncryptor
{
    KeyString = "super-secret-password",
    Padding = AesCbcPadding.Pkcs7,
    Encoder = Base64Encoder.DefaultInstance
};

string cipher = encryptor.EncryptEncoded("payload");
string clear = encryptor.DecryptEncoded(cipher).ToStringUtf8();
```

### RsaKeyPair

Represents a serialized RSA key pair with optional nullable properties at the object model level, but current deserialization requires both keys to be present and non-whitespace.

#### Properties

- `PublicKey`
- `PrivateKey`

#### Serialization behavior

Both properties are written to JSON. `Deserialize(...)` rejects payloads missing either one.

#### Example

```csharp
RsaKeyPair pair = 2048u.CreateRsaKeyPair(RsaKeyPairEncoding.Pem);
string json = pair.Serialize(indented: true);
RsaKeyPair restored = RsaKeyPair.Deserialize(json);
```

### RsaEncryptor

Represents a reusable RSA encryptor/decryptor.

#### Configuration

- `KeyPair` is required
- `Padding` defaults to `OaepSha256`
- `Encoder` defaults to `Base64Encoder.DefaultInstance`

#### Usage flow

1. Supply a valid `RsaKeyPair`
2. Encrypt with the public key
3. Decrypt with the private key

#### Example

```csharp
RsaKeyPair pair = 2048u.CreateRsaKeyPair(RsaKeyPairEncoding.Pem);

var encryptor = new RsaEncryptor
{
    KeyPair = pair,
    Padding = RsaPadding.OaepSha256,
    Encoder = Base64Encoder.DefaultInstance
};

string cipher = encryptor.EncryptEncoded("payload");
string clear = encryptor.DecryptEncoded(cipher).ToStringUtf8();
```
